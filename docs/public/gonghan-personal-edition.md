# 宫寒个人版说明

宫寒小说自动化工作台 / Gonghan Novel Studio 是 Gonghan-Princess 基于上游 AI Novel Writing Assistant 整理的个人 fork。这个版本的目标不是改变原项目的自动导演、章节生产、任务队列或模型接口，而是把公开品牌、首页入口、移动端壳层和新手说明整理成更适合个人持续创作的工作台。

## 版本定位

- 面向长篇小说自动化生产，重点服务“从一句灵感推进到可开写章节”的主链路。
- 保留上游项目来源、AGPL-3.0-only 许可证和 NOTICE / FORK_NOTICE 归属信息。
- 前端个性化优先控制在展示层、文案层和文档层，不改变 API、路由、query key、数据库结构或自动导演状态判断。
- 首页和移动端导航以“今天继续哪本书、下一步点哪里、哪些任务需要处理”为核心。

## AI 辅助内容标注

本个人版整理过程中，部分品牌文案、说明文档、前端交互文案和 UI 打磨方案包含 AI 辅助生成，并经过人工筛选、修改和验证。

向上游提交 Pull Request 或在相关讨论中说明时，应主动标注：

- 本次贡献包含 AI 辅助生成内容。
- AI 辅助内容主要用于文案整理、文档草稿、前端展示层方案和代码修改建议。
- 未引入未声明来源的第三方素材、模型权重、数据集或受额外许可证约束的代码。
- 若后续新增第三方代码、素材、数据、模型输出或其他受许可证约束内容，应补充来源、许可证和必要归属信息。

## 面向维护者的边界说明

这个个人版前端刷新遵守以下边界：

- 不修改后端接口和数据库契约。
- 不修改自动导演任务状态、恢复逻辑、章节生产链路或模型路由语义。
- 不新增运行时依赖。
- 使用现有 React、Tailwind、shadcn 风格组件和 lucide-react 图标体系。
- 移动端优化只调整布局、可读性和入口组织，不改变功能路径。

## 推荐 PR 说明口径

可以在 PR 描述中写明：

> 本 PR 是 Gonghan-Princess 个人 fork 的前端展示与文档整理，主要统一品牌、优化首页创作驾驶舱、增强移动端可读性，并补充 AI 辅助贡献说明。改动保持 API、路由、query key、数据库和自动导演状态判断不变。
